The Problem

A Small Team Drowning in Manual Data Work

A financial services firm managing a portfolio of trading signals needed to monitor performance across three separate platforms — none of which talked to each other. Every day, two team members spent a combined 3+ hours manually pulling data, cross-referencing account balances, checking equity-to-balance ratios, and determining which accounts needed rebalancing.

The process ran twice daily due to the volume of signals, and the manual nature meant critical flags got missed. On weeks when the team couldn't complete all the rebalancing checks on time, higher-risk accounts underperformed compared to their benchmark signals — costing real money.

Before

  • ~3 hours per day of manual data pulling across 3 platforms
  • 2 team members tied up on repetitive extraction work
  • Equity monitoring done by eye — critical rebalancing flags missed
  • When the team fell behind, high-risk accounts underperformed
  • No audit trail — trade-by-trade analysis done ad hoc

After

  • ~4 minutes total runtime for all three scripts daily
  • Both team members freed to focus on strategy and execution
  • Automated equity monitoring flags accounts over 20% threshold
  • Rebalancing alerts delivered same-day, no manual checking
  • Full trade-by-trade audit trail with profit-per-microlot analysis
What I Built

Two Automated Systems From the Same Data Sources

I built two complementary systems that pull from the same three data sources but serve different purposes and different stakeholders within the firm.

System 1: Trade Auditing Pipeline

Audience: CTO & Trading Manager

Automated trade-by-trade analysis and profit-per-microlot calculations across all signals. Detects performance changes and anomalies, and alerts leadership of any deviations. Designed to answer: "Is each signal performing as expected, and if not, what changed?"

System 2: Daily Rebalancing Monitor

Audience: Trading Manager

Automated equity-to-balance monitoring across all live signal accounts. When the difference exceeds 20%, the system flags the account for same-day rebalancing instead of waiting for the weekly cycle. Designed to answer: "Which accounts need attention right now?"

Both systems extract data from three sources that present very different technical challenges:

Data Source 1: Public Performance Feed

Web scraping of signal performance metrics from a public marketplace. Automated extraction on a daily schedule.

Data Source 2: Broker Platform with API Access

Direct API integration for account data and trade history. Batch extraction scripts pull and consolidate data across multiple accounts.

Data Source 3: Legacy Broker Platform (No Standard Access)

This platform actively blocks localhost connections and restricts port access. I built a custom script that runs inside the platform itself and pushes data to a FastAPI server through the one port it allows. This was the hardest technical problem in the project and the piece that made full automation possible.
